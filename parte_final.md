#  Desarrollo de una Biblioteca Virtual en Equipo

## 👥 Trabajo colaborativo

En la parte final de esta práctica trabajamos de forma colaborativa en el desarrollo de una **página web para una Biblioteca Virtual**, utilizando **GitHub** para gestionar el código fuente y coordinar el trabajo entre los integrantes del equipo.

Cada miembro asumió una actividad específica para avanzar de manera organizada y eficiente.

---

# 📝 Organización mediante *Issues*

Para distribuir las tareas, utilizamos la herramienta **Issues** de GitHub, la cual permitió asignar actividades concretas a cada integrante.

Algunos ejemplos de las tareas asignadas fueron:

- 🏠 Desarrollo de la página de inicio (*Home*).
- 🔐 Implementación del formulario de inicio de sesión (*Login*).
- 📚 Creación del formulario de préstamos de libros.
- 📖 Diseño del catálogo de libros.
- 📞 Desarrollo de la sección de contacto.

---

# 🌿 Trabajo con ramas (*Branches*)

Para evitar conflictos en el desarrollo, cada integrante trabajó en una **rama independiente**.

Esto permitió:

- Desarrollar nuevas funcionalidades sin afectar la rama principal.
- Realizar pruebas antes de integrar los cambios.
- Mantener una estructura organizada del proyecto.

---

# 🔄 Sincronización del repositorio

Durante el desarrollo fue necesario mantener el repositorio actualizado mediante diferentes operaciones de Git como:

- `git pull` para obtener los cambios realizados por otros colaboradores.
- `git add` para preparar los archivos modificados.
- `git commit` para registrar los cambios realizados.
- `git push` para subir las actualizaciones al repositorio remoto.

Estas acciones permitieron que todos los integrantes trabajaran con la versión más reciente del proyecto.

---

# 🔀 Integración de cambios

Una vez terminadas las tareas individuales, los cambios fueron integrados al proyecto mediante **Pull Requests**.

Este proceso ayudó a mantener la estabilidad y calidad del desarrollo.

---

# 🚀 Actualizaciones realizadas

Durante el trabajo se implementaron diversas mejoras en la página, entre ellas:

- Incorporación de nuevas secciones.
- Mejoras en el diseño y la interfaz.
- Corrección de errores encontrados durante las pruebas.
- Optimización de estilos y estructura del código.
- Integración de los aportes realizados por todos los colaboradores.

---

# ✅ Conclusión

El desarrollo de la **Biblioteca Virtual** permitió aplicar metodologías de trabajo en equipo utilizando herramientas de control de versiones. La correcta organización mediante **Issues**, el uso de ramas para separar funcionalidades y la sincronización constante del repositorio facilitaron una colaboración eficiente y una integración ordenada del proyecto final.
